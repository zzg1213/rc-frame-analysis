# 框架结构数据集生成流程与原理

## 1. 总体流程

数据集生成由两个阶段组成：

1. 布局生成：选择 1-9 号布局类型，按随机种子采样结构参数，生成节点、梁、柱、楼层、截面和网格数据。
2. 结构分析与配筋：读取布局 JSON，计算荷载和地震作用，抽取 X/Y 向二维框架，求解内力，按组合包络进行梁柱配筋，输出分析 JSON 和图像。

核心目标是形成可复现的结构样本：同一布局类型、同一配置、同一 seed 应得到稳定的布局输入，并可继续生成对应的内力和配筋结果。

## 2. 布局生成

布局生成器负责把抽象结构类型转为可计算的结构模型：

- 按布局类型确定平面组织方式。
- 采样跨数、跨长、楼层数和层高。
- 根据网格坐标生成节点 `N_i_j`。
- 根据相邻节点生成梁 `B_i_j_dir`，其中 `dir=1` 表示 X 向梁，`dir=2` 表示 Y 向梁。
- 按楼层范围生成柱 `C_i_j`。
- 写入梁柱截面、结构参数和几何网格。

输出 JSON 是后续分析的唯一输入，分析阶段不重新推断随机布局。

## 3. 荷载与地震作用

分析阶段首先识别闭合楼板单元：一个网格单元四边均存在梁时，认为该单元存在楼板。由此计算梁的从属宽度：

- X 向梁取上下相邻楼板半跨宽度。
- Y 向梁取左右相邻楼板半跨宽度。
- 边梁因一侧无板，从属宽度自然减少。

随后将楼面恒载、活载、屋面荷载、墙线荷载和梁柱自重换算为构件线荷载与楼层重量。地震作用采用底部剪力法：

- 根据经验周期计算结构基本周期。
- 由地震影响系数得到底部剪力。
- 按楼层高度和重量分配水平地震力。
- 形成楼层地震力和层剪力。

## 4. 二维框架内力分析

三维平面布局被分解为 X 向和 Y 向多榀二维框架：

- 沿每条轴线收集连续梁段。
- 将对应轴线上的柱纳入同一榀框架。
- 对不连续梁段拆分为独立子框架。

每榀框架用矩阵位移法求解：

- 建立梁柱单元局部刚度矩阵。
- 用坐标转换矩阵装配整体刚度矩阵。
- 对竖向荷载、水平地震作用分别组装等效节点荷载。
- 施加底部约束，求解节点位移。
- 恢复构件端弯矩、剪力和轴力。

## 5. 组合包络与配筋

分析结果按恒载、活载和地震作用组合形成内力包络。配筋阶段使用包络控制值：

- 梁按支座负弯矩、跨中正弯矩和端部剪力设计纵筋与箍筋。
- 柱按轴力和弯矩进行偏心受压配筋，并检查轴压比。
- 当轴压比超过限值时，程序增大柱截面并重新分析，直到满足限制或达到最大迭代次数。

最终输出包括：

- 地震参数、层重和层剪力。
- 梁柱内力包络。
- 梁柱配筋面积、实际选筋和验算状态。
- 内力图、层剪力图和楼层配筋图。

## 6. 数据集字段

布局 JSON 主要字段：

- `parameters`：结构类型和楼层数。
- `geometry`：`grid_x`、`grid_y`、`grid_z`。
- `nodes`：平面节点坐标。
- `beams`：各楼层梁信息。
- `columns`：各楼层柱信息。
- `sections`：梁柱截面尺寸。

分析 JSON 主要字段：

- `design_parameters`：材料、荷载和抗震参数。
- `seismic`：周期、地震影响系数、底部剪力、楼层力和层剪力。
- `story_weights`：各层重量。
- `beams`：梁内力包络和配筋。
- `columns`：柱内力包络、配筋和轴压比检查。
